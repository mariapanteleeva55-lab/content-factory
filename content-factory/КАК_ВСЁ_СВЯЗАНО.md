# Как все сервисы работают вместе — полная машина

Представь конвейер на заводе. Каждая станция берёт деталь от предыдущей, что-то с ней делает и передаёт дальше. Наша система устроена точно так же.

---

## Главный принцип

Каждый сервис умеет делать только одно, но делает это очень хорошо.
Наша задача — правильно соединить их в цепочку, чтобы результат одного становился входными данными для следующего.

```
YouTube/TikTok     yt-dlp          Whisper          Claude
  (видео) ──────▶ (скачать) ──────▶ (в текст) ──────▶ (понять)
                                                          │
                                                          ▼
FFmpeg           HeyGen          ElevenLabs           Claude
(склеить) ◀──── (аватар) ◀──── (голос) ◀──────── (написать)
    │
    ▼
Supabase
(сохранить)
    │
    ▼
Браузер
(скачать)
```

---

## Подробная схема — шаг за шагом

### Шаг 0 — Ты нажимаешь кнопку

Ты вставляешь ссылку на видео или выбираешь тему для поиска.
Браузер отправляет это на наш сервер.
Сервер отвечает: «принял, задача №1847, жди».
С этого момента машина работает сама. Ты просто смотришь на прогресс.

---

### Шаг 1 — Поиск (если ищем сами, не по ссылке)

```
Наш сервер
    │
    ├──▶  YouTube API  ──▶  возвращает список: [название, ссылка, просмотры, дата]
    │
    ├──▶  Apify → TikTok  ──▶  возвращает список: [название, ссылка, лайки, скорость роста]
    │
    └──▶  Reddit API  ──▶  возвращает список: [тема, ссылки на обсуждения, голоса]

Сервер берёт все три списка, сортирует по скорости набора просмотров
и показывает тебе топ-20 самых горячих прямо сейчас.
```

**Что передаётся:** текстовые данные — названия, ссылки, цифры.
Никакое видео ещё не скачивается — только метаданные.

---

### Шаг 2 — Скачивание

```
Наш сервер  ──▶  yt-dlp  ──▶  YouTube / TikTok / Instagram
                    │
                    │  скачивает файл (500 МБ – 3 ГБ)
                    │
                    ▼
             Временная папка на сервере
             /temp/task_1847/source.mp4
```

yt-dlp — это не сторонний платный сервис, а программа которая работает прямо на нашем сервере.
Она умеет скачивать с тысячи сайтов, обходя большинство ограничений.

Одновременно вытаскивает отдельно аудиодорожку:
```
source.mp4  ──▶  yt-dlp  ──▶  audio.mp3
```

Аудио нужно для транскрибации — отдельный файл легче и быстрее обрабатывается.

---

### Шаг 3 — Транскрибация (речь → текст)

```
audio.mp3  ──▶  OpenAI Whisper API
                    │
                    │  отправляем файл через интернет на серверы OpenAI
                    │  OpenAI обрабатывает (30–120 секунд)
                    │
                    ▼
             Получаем обратно текст:
             "Привет! Сегодня расскажу почему я перестала
              использовать обычные кремы и перешла на..."
             + временны́е метки каждого слова [00:00:02 - 00:00:08]
```

Временны́е метки пригодятся позже — для автоматических субтитров в финальном видео.

Если аудио длиннее 25 минут — наша система автоматически режет его на куски по 10 минут, отправляет каждый отдельно и склеивает результаты обратно.

---

### Шаг 4 — Анализ вирусного ДНК

```
[текст из Шага 3]
[название видео]
[количество просмотров]
        │
        ▼
    Claude API
        │
        │  мы отправляем:
        │  «Вот текст видео которое набрало 5 млн просмотров.
        │   Проанализируй: какой крючок в начале? какую боль задевает?
        │   какова структура? что заставляет досматривать до конца?»
        │
        ▼
    Claude отвечает структурированным JSON:
    {
      "hook": "провокационный вопрос в первые 3 секунды",
      "pain_point": "страх химических ингредиентов в косметике",
      "structure": ["провокация", "проблема", "решение", "доказательство", "призыв"],
      "emotion": "доверие + страх упустить",
      "viral_score": 8.5
    }
```

JSON — это структурированный формат данных. Не просто текст, а чёткие поля которые машина может читать и использовать дальше.

---

### Шаг 5 — Написание сценария

```
[анализ из Шага 4]
[информация о продукте Мария]
[тон голоса бренда]
[нужный формат: TikTok / Reels / YouTube]
        │
        ▼
    Claude API (второй запрос)
        │
        │  мы отправляем:
        │  «Используя эту структуру и эти триггеры,
        │   напиши сценарий для видео о креме Мария.
        │   Формат: TikTok, 30 секунд, аудитория: женщины 25-40 лет»
        │
        ▼
    Claude возвращает:
    {
      "script": "Ты когда-нибудь читала состав своего крема? [пауза]
                 Я прочитала — и была в шоке...",
      "duration_sec": 28,
      "notes": "интонация — доверительная, как подруга"
    }
```

Для одного видео создаётся 3 варианта сценария — под TikTok, Reels и YouTube Shorts.
Ты выбираешь лучший или берёшь все три.

---

### Шаг 6 — Синтез голоса

```
[текст сценария из Шага 5]
        │
        ▼
    ElevenLabs API
        │
        │  отправляем: текст + ID голоса Марии
        │  (голос настроен заранее под бренд)
        │
        ▼
    Получаем: voice.mp3
    (живо звучащая речь, ~28 секунд)
```

ElevenLabs знает какой голос использовать потому что мы заранее загрузили туда образец голоса и присвоили ему ID. Каждый раз указываем этот же ID — и голос всегда одинаковый, фирменный.

---

### Шаг 7 — Создание видео с аватаром

```
[voice.mp3 из Шага 6]
[ID аватара Марии]
        │
        ▼
    HeyGen API
        │
        │  отправляем: аудио + ID аватара
        │  HeyGen синхронизирует движение губ с голосом
        │  (обработка: 3–10 минут)
        │
        ▼
    Получаем: avatar_video.mp4
    (ведущий говорит под наш голос)
```

---

### Шаг 8 — B-roll клипы (визуальные вставки)

Пока HeyGen генерирует аватар — параллельно запускается другой процесс:

```
[анализ из Шага 4 — ключевые слова]
        │
        ▼
    Runway ML API  ──▶  генерирует 3–5 коротких клипа
    +
    Pexels API  ──▶  ищет подходящие стоковые видео бесплатно

        │
        ▼
    Получаем: broll_1.mp4, broll_2.mp4, broll_3.mp4
```

Эти два процесса (аватар и B-roll) идут **одновременно** — так быстрее.

---

### Шаг 9 — Финальная сборка

Все части собраны. Теперь FFmpeg склеивает их в один файл:

```
avatar_video.mp4  ─┐
broll_1.mp4       ─┤
broll_2.mp4       ─┤──▶  FFmpeg  ──▶  final_video.mp4
broll_3.mp4       ─┤         (монтирует по сценарию,
voice.mp3         ─┤          добавляет субтитры,
субтитры из Ш.3   ─┘          музыку, логотип бренда)
```

FFmpeg работает прямо на нашем сервере — это бесплатная программа, не внешний API.

---

### Шаг 10 — Сохранение и выдача результата

```
final_video.mp4
сценарий (текст)
анализ ДНК (JSON)
        │
        ├──▶  Supabase Storage  ──▶  облако (постоянное хранение)
        │         получаем ссылку: https://storage.../video_1847.mp4
        │
        └──▶  Supabase Database  ──▶  запись в историю:
                  { id: 1847, дата, ссылка, сценарий, статус: готово }

Временные файлы на сервере удаляются:
/temp/task_1847/  ──▶  🗑️ удалено

Браузер получает уведомление: «готово!»
Ты видишь финальное видео и кнопку «скачать».
```

## Схема полной цепочки

```
ТЫ
 │
 │ вставляешь ссылку или тему
 ▼
СЕРВЕР (FastAPI)
 │ создаёт задачу
 ▼
ОЧЕРЕДЬ (Redis)
 │ ставит в работу
 ▼
ВОРКЕР (Celery)
 │
 ├─ [поиск] ────▶ YouTube API ──▶ список видео
 │                Apify/TikTok ──▶ список видео
 │                Reddit API ───▶ список видео
 │
 ├─ [скачать] ──▶ yt-dlp ──────▶ source.mp4 + audio.mp3
 │                                  (на сервере, временно)
 │
 ├─ [текст] ────▶ Whisper API ──▶ транскрипт + тайминги
 │
 ├─ [анализ] ───▶ Claude API ───▶ вирусное ДНК (JSON)
 │
 ├─ [сценарий] ─▶ Claude API ───▶ 3 варианта сценария
 │
 ├─ [голос] ────▶ ElevenLabs ───▶ voice.mp3
 │
 ├─ [аватар] ───▶ HeyGen API ───▶ avatar.mp4     ┐ параллельно
 ├─ [b-roll] ───▶ Runway API ───▶ clips.mp4      ┘
 │
 ├─ [сборка] ───▶ FFmpeg ────────▶ final.mp4
 │                                  (на сервере, временно)
 │
 └─ [сохранить] ▶ Supabase ──────▶ облако + база данных
                                    временные файлы удалены

БРАУЗЕР
 │ каждые 3 секунды спрашивает: «готово?»
 └─ получает: ссылку на скачивание + сценарий + анализ
```

## Что происходит если что-то сломалось

Интернет прервался, сервис не ответил, закончились деньги на API — всё это случается.

Машина устроена так, что каждый шаг можно повторить независимо:

```
Сломался Шаг 3 (Whisper не ответил)?
  → подождать 30 секунд → попробовать ещё раз → если 3 раза не вышло → уведомить

Сломался Шаг 7 (HeyGen завис)?
  → не пересчитывать всё с нуля
  → начать именно с Шага 7, используя уже готовые данные из Шагов 1–6
```

Все промежуточные результаты сохраняются в базу данных после каждого шага. Если что-то упало — не теряем работу.

## Схемы для других типов контента

### Статья

```
Тема  ──▶  Claude  ──▶  готовый текст  ──▶  редактор в браузере  ──▶  Supabase
```
Всего один внешний сервис. Самый простой тип контента.

---

### Пост для Instagram / Telegram

```
Тема или продукт  ──▶  Claude  ──▶  3 варианта поста  ──▶  ты выбираешь  ──▶  Supabase
```

---

### Карусель Instagram

```
Тема
  │
  ▼
Claude  ──▶  текст для 8 слайдов (заголовок + 2-3 строки каждый)
  │
  ▼
Шаблонизатор  ──▶  накладывает текст на шаблоны в цветах бренда Мария
  │
  ▼
8 картинок .jpg  ──▶  Supabase  ──▶  скачать архивом
```

---

### Длинное видео YouTube

```
Тема / вирусное видео-референс
  │
  ├──▶  [Шаги 1–5 как обычно]  ──▶  подробный сценарий с таймингами
  │
  ├──▶  ElevenLabs  ──▶  голос (5–15 минут аудио)
  │
  ├──▶  HeyGen  ──▶  аватар (длинная версия)
  │
  ├──▶  Runway + Pexels  ──▶  много B-roll клипов
  │
  └──▶  FFmpeg  ──▶  финальное видео 5–15 минут
```

---

## Важный момент — данные никуда не утекают

Все видео которые мы скачиваем — только для анализа. Мы не храним чужие видео.
После того как текст извлечён — исходник удаляется с сервера автоматически.
В облаке хранится только наш контент: готовые ролики Марии, сценарии, тексты.
