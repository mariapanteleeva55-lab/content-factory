# Инфраструктура — что нам потребуется

Здесь расписано: какие сервисы нужны, сколько стоят, где хранить видео и как вся система устроена.

## Часть 1 — Внешние сервисы и API

Это платформы, к которым наша система подключается. Мы ничего не строим сами — берём готовое.

---

### Искусственный интеллект

**Claude (Anthropic)**
Главный мозг системы. Анализирует видео, пишет сценарии, статьи, посты, тексты для каролелей.
Оплата: за количество слов — примерно $0.01–0.05 за одну задачу (анализ видео или написание сценария).
→ Регистрация: console.anthropic.com

**Whisper (OpenAI)**
Переводит речь из видео в текст. Понимает 99 языков.
Оплата: $0.006 за 1 минуту аудио. Видео на 10 минут = $0.06.
→ Регистрация: platform.openai.com

### Создание видео

**ElevenLabs**
Синтез голоса — превращает текст в живую речь на русском языке.
Можно создать фирменный голос бренда Мария, если записать 10–15 минут своего голоса.
Тариф: от $5/мес (30 минут готового аудио), $22/мес (100 минут).
→ Регистрация: elevenlabs.io

**HeyGen**
ИИ-аватар — виртуальный ведущий, который читает сценарий на камеру.
Выглядит как живой человек: двигает губами, моргает, жестикулирует.
Можно загрузить своё видео и сделать цифрового двойника.
Тариф: от $29/мес (базовый), $89/мес (профессиональный с клонированием).
→ Регистрация: heygen.com

**Runway ML**
Генерирует короткие видеоклипы по описанию — для B-roll вставок в видео.
Описываешь: «крем наносится на кожу, крупный план, тёплый свет» — сервис создаёт клип.
Тариф: от $12/мес (125 секунд видео), $28/мес (625 секунд).
→ Регистрация: runwayml.com

### Поиск вирусного контента

**YouTube Data API**
Официальный поиск по YouTube. Ищем видео по темам косметики, ухода за кожей, трендов.
Стоимость: бесплатно (лимит 10 000 запросов в день — нам хватит).
→ Получить ключ: console.cloud.google.com

**Apify**
Сервис для сбора данных из TikTok (TikTok сам не даёт официального доступа).
Находим видео, набирающие просмотры прямо сейчас.
Тариф: от $49/мес, но нам достаточно Pay-as-you-go — примерно $5–15/мес при нашем объёме.
→ Регистрация: apify.com

**Reddit API**
Форумы где люди обсуждают косметику, задают вопросы, делятся советами.
Стоимость: бесплатно.

## Часть 2 — Где хранить видео

Это важный вопрос. Видео весят много — один исходник с YouTube может занимать 500 МБ — 3 ГБ.
На личном компьютере это не решить — места не хватит, и система будет работать только пока компьютер включён.

### Как это будет устроено

Система работает в двух режимах хранения:

**Временное хранение (на сервере)**
Пока идёт обработка — видео живёт на сервере. Это как рабочий стол: скачали, порезали, транскрибировали, проанализировали — и сразу удалили исходник.
Весь этот процесс занимает 5–15 минут. После него исходное видео больше не нужно.

**Постоянное хранение (в облаке)**
Готовые материалы — финальное видео, сценарии, анализ — сохраняются в облачное хранилище.
Туда же попадают готовые карусели, статьи и посты.
Из облака их можно скачать в любой момент.

### Варианты облачного хранилища

**Cloudflare R2** *(рекомендуем)*
Очень дёшево: $0.015 за 1 ГБ в месяц.
Нет платы за скачивание файлов — это важно, у других сервисов это стоит дополнительно.
100 ГБ видео = $1.5/мес. Очень выгодно.

**Supabase Storage**
Бесплатно до 1 ГБ, затем $0.021 за ГБ.
Удобно тем, что в Supabase же будет и база данных — всё в одном месте.
Хороший вариант для старта.

**Amazon S3**
Самый известный, надёжный, но дороже.
$0.023/ГБ + платишь за каждое скачивание.
Имеет смысл если вырастем до большого объёма.

→ **Наш выбор для старта:** Supabase (база данных + хранилище вместе, просто настроить)

## Часть 3 — Сервер

Система не будет работать на личном компьютере по трём причинам:
1. Компьютер нужно держать включённым 24/7
2. Домашний интернет медленный для скачивания видео
3. Тяжёлые задачи (монтаж, обработка) будут тормозить всё остальное

Нам нужен **VPS** — виртуальный сервер в дата-центре. Это компьютер в интернете, который работает всегда.

### Что нужно серверу

| Ресурс | Минимум | Комфортно |
|--------|:-------:|:---------:|
| Процессор (CPU) | 2 ядра | 4 ядра |
| Оперативная память (RAM) | 4 ГБ | 8 ГБ |
| Диск (временное хранение) | 20 ГБ | 50 ГБ |
| Интернет | 100 Мбит/с | 1 Гбит/с |

Диск нужен только для временного хранения пока идёт обработка — готовые файлы уходят в облако.

### Где взять сервер

**Railway** *(рекомендуем для старта)*
Очень просто настроить — подключаешь GitHub и система сама разворачивается.
Цена: от $20/мес за нужные мощности.
Плюс: не нужно разбираться в серверном администрировании.

**Render**
Похоже на Railway, тоже простой старт.
Цена: от $25/мес.

**DigitalOcean / Hetzner** *(для роста)*
Чуть сложнее настроить, но дешевле.
Hetzner (немецкий) — 4 ядра, 8 ГБ RAM, 80 ГБ диск = **€8.5/мес** (очень выгодно).
DigitalOcean — аналогичная конфигурация = $24/мес.

→ **Наш выбор для старта:** Railway (просто) → потом Hetzner (дёшево)

## Часть 4 — Архитектура системы

Это схема того, как все части соединены между собой.

```
┌─────────────────────────────────────────────────┐
│                  ТВОЙ БРАУЗЕР                   │
│         (открываешь сайт, нажимаешь кнопки)     │
└───────────────────┬─────────────────────────────┘
                    │ запросы
                    ▼
┌─────────────────────────────────────────────────┐
│                   СЕРВЕР (VPS)                  │
│                                                 │
│  ┌─────────────┐    ┌──────────────────────┐   │
│  │  Веб-сервер │    │   Очередь задач      │   │
│  │  (FastAPI)  │───▶│   (Redis + Celery)   │   │
│  │             │    │                      │   │
│  │ принимает   │    │ хранит задачи,       │   │
│  │ запросы и   │    │ раздаёт воркерам     │   │
│  │ отдаёт      │    └──────────┬───────────┘   │
│  │ результаты  │               │               │
│  └─────────────┘               ▼               │
│                    ┌──────────────────────┐    │
│                    │      ВОРКЕР          │    │
│                    │  (фоновый процесс)   │    │
│                    │                      │    │
│                    │ 1. скачивает видео   │    │
│                    │ 2. транскрибирует    │    │
│                    │ 3. анализирует       │    │
│                    │ 4. пишет сценарий    │    │
│                    │ 5. генерирует видео  │    │
│                    └──────────┬───────────┘    │
└───────────────────────────────┼────────────────┘
                    │           │
          временные файлы       │ готовые файлы
          (видео, аудио)        │
                    │           ▼
                    │  ┌─────────────────┐
                    │  │  ОБЛАКО         │
                    │  │  (Supabase /    │
                    │  │  Cloudflare R2) │
                    │  │                 │
                    │  │  - готовые видео│
                    │  │  - сценарии     │
                    │  │  - статьи       │
                    │  │  - карусели     │
                    │  └─────────────────┘
                    │
                    ▼  (удаляются после обработки)
                 🗑️ исходное видео удалено
```

### Что происходит когда нажимаешь кнопку

1. **Браузер** отправляет запрос на сервер: «обработай это видео»
2. **Сервер** получает запрос и кидает задачу в очередь: «поставить в работу»
3. **Воркер** берёт задачу из очереди и начинает работать:
   - Скачивает видео с YouTube/TikTok на сервер
   - Вытаскивает звук и отправляет в Whisper → получает текст
   - Отправляет текст в Claude → получает анализ вирусного ДНК
   - Снова в Claude → получает готовый сценарий
   - Отправляет сценарий в ElevenLabs → получает аудиофайл с голосом
   - Отправляет в HeyGen → получает видео с аватаром
   - FFmpeg склеивает всё в один финальный ролик
   - Готовый ролик отправляется в облако
   - Исходное видео удаляется с сервера
4. **Браузер** каждые 3 секунды спрашивает сервер: «готово?»
5. Когда готово — показывает результат, даёт ссылку на скачивание

### База данных

Все результаты сохраняются в базу данных (Postgres):
- Какие видео анализировали
- Какие сценарии написали
- Где лежит готовый ролик в облаке
- Когда создали, какой статус

Это нужно для истории — чтобы можно было вернуться к любому старому проекту.
